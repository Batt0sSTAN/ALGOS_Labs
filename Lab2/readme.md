## Браузерная история (Browser History)
### Описание задачи
Необходимо реализовать систему хранения истории браузера. Каждая запись содержит:

- URL
- время посещения
- флаг "закладка"
- Обязательная часть

Реализовать:

- Навигацию "назад / вперед"
- Очистку истории
- Поиск по домену

Вариативная часть:
- Вывод истории в виде таблицы с выравниванием колонок
- Топ-N переходов (наиболее частые переходы между доменами)

## Обоснование структур данных

Для хранения всей истории используется динамический массив
Для отдельной записи в истории браузера используется двусвязный 

- Двусвязный список — идеален для навигации «назад/вперед».Перемещение по истории происходит за O(1), так как каждый узел хранит ссылки на предыдущую и следующую записи.При переходе на новый URL с «промежуточной» позиции (когда мы вернулись назад), удаление старой «ветки» вперед и добавление нового элемента также происходит за O(1) (изменением указателей), в отличие от динамического массива, где удаление элементов из середины/конца требует сдвига или изменения размера за O(n).

- Обеспечивает добавление в конец за амортизированное O(1).Позволяет эффективно фильтровать данные при поиске по домену и при подсчете топ-N переходов.
